Accessing Xcalar Design

Before accessing Xcalar Design, make sure that the following requirements are met:

  • You use one of the following web browsers:

    • Google Chrome version 65 or later
    • Microsoft Edge version 44 or later

    Also, Xcalar Design does not run on mobile devices such as tablets or cell phones.

  • The source files containing the raw data to be analyzed are accessible to Xcalar Design.

The minimum recommended screen resolution for Xcalar Design is 1024 x 768.

Starting Xcalar Design

To start Xcalar Design, point your browser to the URL provided by Xcalar. Depending on your previous Xcalar Design login history, one of the following will apply:

  • If you are already logged into Xcalar Design:

    The Xcalar Design Workbook Browser page is displayed and you can continue using Xcalar.

  • If you already have a Xcalar Design account but you are logged out:

    • In the Login page, enter your email address and password, and click Log In.

      If the login is successful, the cluster size selection page is displayed.

    • If you do not remember your password:

      1. Click Forgot password.

        The Password recovery page is displayed.

      2. Enter your email address and click Continue.

        An email with a verification code is sent to your email address and the Choose new password page is displayed in your browser window.

      3. Enter the code from the email and a new password in the Choose new password page. Reenter the password in CONFIRM THE PASSWORD field.

      4. Click Reset Password.

        The new password is saved and the cluster size selection page is displayed.

  • If you do not have a Xcalar Design account:

    1. In the login page, click Create an account.

      The sign up page is displayed.

    2. Enter your first name, last name, company name, email address and a password. Reenter the email address and the password to confirm.
    3. Read and accept the End User License Agreement (EULA).

    4. Click Create account.

      An email is sent to you with a verification code and the Verify your account page is displayed for you to enter that verification code.

      NOTE: If you do not receive the email in your Inbox within a few minutes, check your Spam folder. If you donâ€™t see the email in the Spam folder either, click Didn't receive a code? Resend it in the Verify your account page for the verification code email to be resent.

    5. Enter the verification code in the Verify your account page and click Confirm account.

      The account is verified and a cluster size selection page is displayed.

Selecting a cluster size

Selecting a cluster size on every login gives you the flexibility to experiment with different cluster sizes. Your existing data models are not affected by selecting a different cluster size.

Select a cluster size depending on your needs as described in Understanding Xcalar cluster sizes and click DEPLOY. Xcalar starts the process of bringing your cluster up, which may take a few minutes. Once the cluster is up, the Xcalar Design Workbook Browser page is displayed and you can start using Xcalar.

Understanding Xcalar credits

When you sign up for Xcalar Cloud, you are granted certain number of credits. The credits get consumed depending on the cluster size you select and the duration for which you run the cluster. Credits are consumed only if your Xcalar cluster is running. For information on how to stop a cluster, see Logging out of Xcalar Design .

The number of remaining credits are displayed under the User icon in the top right corner of Xcalar Design. Once the credits go low, you get a warning message, but you can continue using Xcalar. If all your credits are consumed, you cannot use the product any more unless new credits are added to the system. To add credits to your account, click the User icon and then the Add link next to the credits.

Logging out of Xcalar Design

When you don't need to use Xcalar Design, it is recommended that you log out of Xcalar Design explicitly. To log out of Xcalar Design, click the User icon in the top right corner of Xcalar Design and select Log Out. A modal is displayed for you to select if you want to keep the cluster running. If you have a data model executing, you can log out from the UI but leave the Xcalar cluster running so that your data model continues to run in the background. Otherwise, you should select to stop the cluster and log out.

Xcalar Design also provides a setting to control the idle duration after which the user is automatically logged out and the idle cluster is automatically stopped. For more information on this setting, see Configuring general settings.

You can have multiple logins at any given time. For example, after logging in from one browser tab, you can log in again from another browser tab. However, you cannot open the same workbook simultaneously from multiple browser tabs.

Suppose you are logged in to Xcalar Design from two browser tabs. After you open a workbook in one browser tab, your attempt to open the same workbook in another browser tab generates a warning message. You can either go back to the previous screen or continue opening the workbook in the second browser tab. If you continue, the workbook in the first browser tab is closed.

Effects of stopping your Xcalar cluster

Following are the effects of stopping your Xcalar cluster:

  • The credits are not consumed for the time your Xcalar cluster is not running.
  • All the datasets and tables in your workbooks are deactivated. After logging back in, if you try to use a dataset or a table, you may have to activate it first. However, when you execute the dataflows, the required tables and datasets are automatically activated.
  • The intermediate results of all Nodes in a dataflow are dropped.
