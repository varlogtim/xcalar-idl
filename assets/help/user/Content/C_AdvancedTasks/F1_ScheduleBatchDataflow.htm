Scheduling batch dataflows

After you create a batch dataflow, you can run the dataflow at regular intervals. (This feature is not available in Xcalar Cloud Preview.)

NOTE:  When you manually run a batch dataflow, the result can be an exported file or table. However, running a batch dataflow automatically based on a schedule can only result in an exported file.

Preparing a batch dataflow to run on schedule involve two steps, which are parameterizing the export file name and creating a schedule.

Parameterizing the export file name

You must parameterize the export file name so that repeated runs of the batch dataflow do not overwrite the previous version of the export file. If Xcalar Design detects that an export file of the same name already exists, the batch dataflow fails to run.

Follow these steps to parameterize the export file name:

  1. In the dataflow list, click the name of the batch dataflow to display it.
  2. Right click the export file icon (last icon) of the batch dataflow to display a pop-up menu.
  3. Click Create Parameterized Query in the pop-up menu.

  4. If the Export As field is not filled in, click the Default value button to automatically add the default export file name in the field as shown in the following screenshot.

  5. In the Parameterize Query window, click and drag the N parameter to the file name shown in the Export As field. This parameter is a system parameter. By default, its value starts at 0 and is incremented by 1 each time the query is run.

    EXAMPLE: If the export file name is export-airlines#XS366.csv, you can drag the N parameter to the name so that the name becomes export-airlines<N>#XS366.csv. Running the batch dataflow the first time creates a file named export-airlines0#XS366.csv, running it the second time creates export-airlines1#XS366.csv, and so on.
  6. Click SAVE.

For more information about parameterization, see Parameterizing queries.

Specifying a schedule

Follow these steps to create a schedule for a batch dataflow:

  1. In the dataflow list, click for the selected batch dataflow as illustrated in the following screenshot.

  2. You can specify a schedule in either simple mode as described in this step or in advanced mode as described in step 3. (Advanced mode is recommended only for users familiar with the cron utility on UNIX systems.)

    Specify the following information in the Create New Schedule panel. See the screenshot following the list for more information about how to specify the information.

    • The start time through a calendar and a 12-hour clock, which determines the first time the batch dataflow will be run. The time specified is based on the server time, not your browser time. The time specified must be in the future.
    • How often the batch dataflow is run.

    Go to step 4.

  3. To specify a schedule in advanced mode, enter a cron expression in the Schedule field. Then click SIMULATE. The Last Run and Next Run fields display the times when two consecutive runs can take place. These times help you visualize the interval between any two runs if the batch dataflow is executed according to your schedule.

    IMPORTANT: The cron expression in this window allows only asterisk, digit, forward slash, comma, and hyphen. Do not use other characters even though they are valid on other UNIX systems.

    In the following example, the batch dataflow runs every minute, every day, starting at 4 p.m. until 4:59 p.m.

  4. Click SAVE. This schedule is now associated with the batch dataflow.
NOTE: After you create a schedule, you cannot modify it. If you want to apply a different schedule to the batch dataflow, delete the schedule and create a new one.
IMPORTANT: The batch dataflow can run only if the export file name is parameterized. If you have not parameterized the name already, follow the instructions in Parameterizing the export file name.

Deleting a batch dataflow schedule

Follow these steps to delete a batch dataflow schedule:

  1. In the dataflow list, click for the selected batch dataflow.

  2. Click the Delete icon as shown in the following screenshot.

  3. When the confirmation message is displayed, click CONFIRM.
