Glossary

The following table is a Xcalar glossary of terms.

Term Meaning
Batch dataflow You can save a dataflow under a name. A named dataflow is a batch dataflow, which can be scheduled to run at pre-set times. Running a batch dataflow lets you run operations against the entire data source unattended as opposed to executing individual operations interactively against sampled data. It also enables you to create a table based on the dataflow with selected columns. You can then export the table to another application or the active worksheet in Xcalar Design. A batch dataflow goes through optimization in a way similar to how a SQL query goes through query optimization. However, neither dataflow nor query optimization can guarantee optimal performance, nor necessarily meet Service Level Objectives (SLO) for real-world workloads. To achieve that, you need to operationalize the batch dataflow. For information about operationalizing dataflows, see Operationalizing batch dataflows
Dataflow A set of elementary operations that shows the data lineage for the active table. Xcalar Design displays a dataflow by using ovals and rectangles to represent tables and operations, respectively.
Dataset The representation of raw data and metadata that Xcalar creates when a user points to a data source. The dataset may contain all data or a sample of the data in the data source, depending on the dataset size limit that the user specifies when pointing to a data source.
Dataset reference When you point to a data source, Xcalar creates a dataset. Xcalar Design also creates an icon that provides you with a mechanism to access the dataset. The icon is called a dataset reference because it references a dataset, enabling you to preview the dataset and create tables from the dataset. It is possible to have a dataset reference without a dataset. For example, when you try to point to a data source but the pointing fails due to various reasons such as the dataset size limit being exceeded, the dataset reference is created nevertheless. This dataset reference simply gives you a convenient way to re-point to the data source.
Data source The file containing raw data or the directory containing files with raw data that is being pointed to by XCE.
Drop a table Dropping a table means completely removing the table (not just its contents) from a worksheet (for example, to conserve memory). This action is not reversible. It is not equivalent to "delete a table" in SQL.
Export This is the process for copying a table in Xcalar Design (for example, a resultant table created by an operationalized dataflow) to a location for another application to use.
Function A function defined by Xcalar is an XDF (Xcalar defined function) and a function defined by the user is a UDF (user defined function). You execute a function to either transform or extract data (for example, to find the average, to filter, and so on).
Modeling

One or all of these tasks:

  • synthesizing a schema
  • designing an algorithm
  • designing a query
  • designing a linear regression analysis
  • designing a new machine learning algorithm

Xcalar will expand this list of supported modeling tasks in future releases.
Node The physical or virtual execution environment that is a member of a cluster, for example, a server or a virtual machine running Linux.
Operation A database operation, which is an overall process to accomplish a transformation. For example, "group by" is an operation. An operation can include one or multiple functions. For example, a map operation can contain a function that means, "If a value in column X is greater than 100, create a new column called Bad-percent, which is True, else set value in new column to False."
Operationalizing batch dataflows For Xcalar to meet SLOs (Service Level Objectives), customers need to contact Xcalar professional services who will do performance tuning on their batch dataflows. These batch dataflows, after being tuned, are called operationalized batch dataflows.
Point to data The first thing you do when you use XCE to demystify your data. It means using Xcalar Design to create the dataset reference, which lets you preview the dataset and create tables from the dataset.
Query A request for information from a database. You need the information to answer your particular question.
Sample Collection of records from the data source on which you can interactively model your dataflow.
Session A session is the same as a workbook in Xcalar Design. It does not refer to a user's login. Sessions are visible only to the cluster administrator when the administrator views information about workbooks on the XCE nodes.
Xcalar Compute Engine (XCE) The engine that does the computing. Users interact with XCE through Xcalar Design.
Xcalar Data Platform (XDP) Xcalar product consisting of these components: Xcalar Design, and Xcalar Compute Engine.
Xcalar Design The graphical user interface that users use to do interactive data modeling.
XCE server (or Xcalar Compute Engine server) An instance of XCE.
