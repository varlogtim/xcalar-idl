Working with IMD in Xcalar Design

This section describes how to work with IMD in Xcalar Design for analyzing data that is continuously changing.

Working with mutable tables

IMD stands for insert, modify (update), and delete, which are commonly used data manipulation operations for making changes to data. IMD enables you to process, track changes, and view the data changes from these operations on your most current and historical temporal data for comparison and analysis.

In Xcalar Design, a Published Table supports the data manipulation operations, as well as, tracks data changes, enabling you to revert to any data state in a Published Table at any given time.

The IMD extension inside Xcalar Design creates the following:

  • A base table from your Xcalar table known as a Published Table.
  • A delta table from a Xcalar table known as an Update Table.

Combined, these tables store your data updates in the Published Table for later viewing and analysis.

A new Xcalar table is also generated from the delta table for viewing the results of the delta table data changes in your Published table.

Your Published tables are listed in the IMD screen under the Published Tables section. From the IMD screen you can instantly see when changes were applied and compare and analyze changes on all your data or from a specific point-in-time.

Though Xcalar Published tables are generally updated as part of a Xcalar batch dataflow pipeline that trigger data changes in real-time, you can also update Published tables from standalone datasets or multiple datasets that contain data timestamps.

Working with the IMD extension

The IMD extension enables you to create your base and delta tables. This topic describes how to create a base table known as Published Table and a delta table known as an Update Table.

Understanding the Xcalar data manipulation column action values

The base table and update tables must contain a data manipulation action type column, known as the opcode column, for tracking which action type was performed during the change. You can create a data manipulation action type column with the Xcalar values below, from a user-defined function (UDF).

The opcode column must contain the following Xcalar data manipulation action type values:

  • 0, which represents row that has been deleted
  • 1, which represents a row that has been inserted
  • 2, which represents a row that has been updated.

If you have a data manipulation action type column, such as an opcode column created by change data capture (CDC) software, you must verify that the values are identical to those of the Xcalar data manipulation action type.

To update incorrect values in your opcode column, use the MAP function in Xcalar Design.

Creating the IMD Published and Update tables

Pre-requisites: When creating a Xcalar table for either your base Published Table or your delta Update table, you must do the following:

  • (Optional) Include a File Name and a Record Number column. For more information, see Setting advanced options.
  • Verify that the Xcalar table contains a column that can be used as a primary key identifier. This column identifies your data records and is used to track changes.
  • (Optional) If your Xcalar table contains an order by column, run a Sort operation.
  • (Optional for a Published Table) Contains an opcode column.

Creating a Published Table

This topic describes how to create a Xcalar Published Table from a Xcalar table, which is your base table for storing changed data values.

To create a Published Table:

  1. From the IMD extension, select Publish Table.

    2. The IMD: Publish Table section appears.

  2. From the Table list, select the Xcalar table which will be used to create a base table.
  3. In the Published Table Name field, enter the name for the Published Table.
  4. In the Primary Key field, enter the name of the column that contains your primary key values. For example, the name you gave to the File Name column.
    5. TIP: In your Xcalar table, click on the column name, which populates the column name into the Primary Key field.
  5. In the IMD Operator field, enter the name of the opcode column.
    6. IMPORTANT: You must verify that the opcode column values are identical to those of the Xcalar data manipulation action type, for more information, see Understanding the Xcalar data manipulation column action values
    IMPORTANT: If no opcode name is entered in the IMD operator field, Xcalar assumes that all the data changes are inserts and populates all rows in the XcalarOpCode column with a value of 1 (insert).
  6. Click APPLY, which does the following:
    • Creates a Published Table containing derived fields.
    • Adds two columns named XcalarRankOver and XcalarOpCode, which are Xcalar metadata columns.

The Published Table is now prepared for receiving data updates.

Creating an Update Table

This topic describes how to create a Xcalar Update Table from a Xcalar Table, which is your delta table that is used for applying the latest update values to the Published Table.

IMPORTANT:
- Your delta table cannot contain columns that were not in the original Published Table. If it does contain columns that were not in the original Published Table, hide the columns before applying the Update Table option of the IMD extension.
- Your data table must contain an opcode column. If no opcode name is entered in the IMD operator field, Xcalar assumes that all the delta table data changes are inserts.

To create an Update Table:

  1. From the IMD extension, select Update Table.

    2. The IMD:Update Table section appears.

  2. From the Table list, select the name of the delta table that contains the new data updates.
  3. In the Name of Published Table field, enter the name of the Published Table for applying this delta tableâ€™s updates.
  4. In the Primary Key field, enter the name of the primary key column.
  5. In the IMD operator field, enter the name of the opcode column.
  6. Click APPLY, which does the following:
    • Creates an Update Table.
    • Applies the delta table data updates to the Publish table.
    • Creates a new Xcalar table from the Update table, which is used for viewing the results of the delta table data changes in the Published table.

Working with the IMD screen

This topic describes how to work with the IMD screen to view and analyze your mutable data.

Displaying the Xcalar Design IMD screen

From the Xcalar Design sidebar, click the IMD icon , which displays the IMD screen. The following image describes the major elements in the IMD screen.

Understanding the IMD screen

The Xcalar Design IMD screen contains two sections for viewing and working with your Published Tables:

  • Published Tables, which list your current Published tables. Time-bar lines show when the Published table was created and when data updates occurred.
  • Hidden Tables, which list the Published tables that you are currently not working with. For example, you can hide specific Publish tables when they are not required as part of a Point in time or Latest action. Hidden Published tables are not viewable on the IMD timeline.
    • NOTE: Data changes are still applied to Hidden tables.

The following table describes the screen elements:

Table 1 IMD elements
Element Description

Date-picker

This field, which is horizontal with the Xcalar Design Browser icon, enables you to display within the IMD screen a specific date range.

Either enter a date using the MM/DD/YYYY format or select a date from the calendar element.

Time-picker

This field, which is to the right of the date-picker, enables you to display the specific time based on a 12-hr clock.

Either enter a time using the HH:MM:SS format or from the time-picker element, select the plus or minus buttons to increase or decrease the time.

Timeline bar

 

 

This element, which is horizontal with the Published Tables name, displays the date in increments of hrs, days, weeks, and months.

To zoom in and out use the scroll wheel on your mouse.

Hovering your mouse over the Timeline bar displays a tool tip with the exact date and time.

Time-bar

This element controls actions across all visible Published tables, such as viewing data from a specific point-in-time.

Table pop-up

This element, opens when you click anywhere in the Published Tables section of the IMD timeline screen.

The table pop-up contains the following action buttons:

  • Point in time, which when clicked, generates a Xcalar table for each of the visible Published Tables comprising all data changes up to that point-in-time.
  • Latest, which when clicked, generates a Xcalar table for each of the visible Published Tables comprising all data changes up to the current time.
Scroll bar

This element, which is below the Published Tables section, enables you to scroll along the IMD timeline.
Details section

This section contains more details about your Published Table.

For more information, see Understanding the Details section.

Hide published table icon

Clicking on this icon moves the Published Table to the Hidden Tables section.

Show published table icon

Clicking on this icon moves the Published Table to the Published Tables section.
Trash can icon

Deletes the Published Table.

WARNING: When you delete a Published Table, you can no longer view or apply updates to the Published table or view any tables generated from the Published table and their descendant tables. Xcalar recommends that you either do not delete a Published table or that before you delete a Published table you generate a Latest table from the Table pop-up. This ensures that if for any reason you need to continue to use the data that produced the Published table you have a Xcalar table for creating a new Published table. Any new updates from the time you deleted the Published table will fail.

Generating Xcalar tables from multiple Published Tables

You can generate a worksheet that contains Xcalar tables from all the Published tables in the Published Tables section, where the data comprises all changes up to a specific point-in-time or all changes up to the current time.

To generate Xcalar tables from multiple Published tables:

  1. From the Xcalar Design sidebar, select the IMD icon .

    2. The IMD screen opens.

  2. For each Published table not required for this action, from the Published Tables section, click the Hide publish table icon.

    3. The Published table is moved to the Hidden Tables section.

  3. Do one of the following:
    • For a Xcalar worksheet that contains Xcalar tables that comprise all data changes up to the specific point-in-time, move the time-bar to the specific date, click, and from the Table pop-up, click Point in time.
    • For a Xcalar worksheet that contains Xcalar tables that comprise all data changes up to the current time, click anywhere in the IMD timeline screen, and from the Table pop-up, click Latest.
  4. From the Worksheets icon, open the worksheet that contains the new Xcalar tables for analysis.
