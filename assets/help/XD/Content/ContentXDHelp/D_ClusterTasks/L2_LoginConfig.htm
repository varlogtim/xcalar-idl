Configuring users (admin only)

This section describes how to configure user authentication through the LOGIN CONFIGURATION modal window in Xcalar Design.

Displaying the Login Configuration modal window

To display the Login Configuration modal window:

  1. Log in to Xcalar Design as either the Xcalar Admin user or the default admin user.
  2. From the sidebar, click the System icon and then click the Administration icon .
  3. In the Administration panel, click Login Configuration.

The rest of this topic discusses the tasks that you can perform from the Login Configuration modal window. After filling out the required information, click Save. Your changes will take effect immediately.

Enabling or disabling an authentication method

According to the platform on which you installed Xcalar, the following user authentication services are supported:

  • For an on-premises Xcalar deployment, Xcalar Design supports the Lightweight Directory Access Protocol (LDAP) user authentication for verifying user access.
NOTE: During the Xcalar installation, you can choose to configure your LDAP settings before or after the installation. If you choose to configure your LDAP settings after installing Xcalar, you must create a default admin user account during installation by selecting the CREATE DEFAULT ADMIN USER check box.
IMPORTANT: Xcalar recommends creating a default administrator, as this account's credentials are stored locally on your cluster and can be used when the LDAP server is not available.
  • For a Microsoft Azure Marketplace Xcalar cloud deployment, Xcalar Design supports Azure Active Directory v2.0 user authentication or LDAP for verifying user access.
    • NOTE: Xcalar recommends using Azure Active Directory v2.0 for user authentication.

In the Login Configuration modal window, you can enable or disable LDAP, Azure AD authentication by selecting or deselecting its check box. For more information on how to fill out the required field settings, see either Enabling LDAP authentication or Enabling Azure AD v2.0 Common Endpoint authentication.

IMPORTANT: You must enable at least one authentication method. If both LDAP and Azure AD are disabled, users cannot access Xcalar Design. Creating a default admin user during the Xcalar installation, grants administrator user access privileges to Xcalar Design outside of the LDAP and Azure AD authentication method. This user is then able to enable LDAP or Azure AD authentication. For more information about the default admin user account, see Understanding the default admin user account. If you cannot use the default admin user account, call Xcalar for information about accessing your cluster again.

Enabling LDAP authentication

LDAP authentication requires values that were collected during the Xcalar pre-installation tasks. For more information, see the Xcalar Installation Guide.

To enable LDAP Authentication:

  1. Log in to Xcalar Design as either the Xcalar Admin user or the default admin user.

  2. From the sidebar, click the System icon and then click the Administration icon .
  3. In the Administration panel, select Login Configuration.

    4. The LOGIN CONFIGURATION modal window appears.

  4. Select the ENABLE LIGHTWEIGHT DIRECTORY ACCESS PROTOCOL (LDAP) AUTHENTICATION check box and then select your current LDAP implementation.

  5. In the LDAP URL field, enter the LDAP URL that is used to contact the LDAP Server.

  6. In the USER DN field, enter the location in the LDAP server in which to search for user entries. For example,

    7. cn=Users,dc=int,dc=xcalar,dc=net

  7. In the SEARCH FILTER field, enter the LDAP filter that is used to limit the returned number of user entries to one. For example,

    8. (&(objectclass=user)(userPrincipalName=%username%))

  8. In the SERVER KEYFILE field, enter the path to the LDAP security certificates that identify the server.
  9. (AD option only) In the AD USER GROUP field, enter the name of the Active Directory's user group. For example, Xce User.
  10. (AD option only) In the AD ADMIN GROUP field, enter the name of the Active Directory's administrator group. For example, Xce Admin.
  11. (Optional AD option only) In the AD DEFAULT DOMAIN field, enter the domain name of your Active Directory, which is used as an optional default domain name.
  12. (Optional AD option only) If your Active Directory's top level groups contain sub groups, select the ENABLE GROUP SUBTREE SEARCHING check box. This enables subgroup LDAP searching for AD LDAP.
  13. (Optional AD option only) By default, when a user enters their user name and password to log in to Xcalar Design, Xcalar authenticates the user by searching AD for both a user name and the AD domain name. If you require user authentication by Xcalar based on the user name only, select the SEARCH USING USER ID ONLY check box.
  14. If you require secure LDAP connections, select the ENABLE TRANSPORTATION LAYER SECURITY (TLS) check box.
  15. Click Confirm & Save.

Enabling Azure AD v2.0 Common Endpoint authentication

The Azure AD authentication fields require Azure AD Xcalar resource identifiers. The following topics describe how to collect the identifiers and enable Azure AD authentication.

Recording Azure AD Xcalar resource identifiers

To record Azure AD Xcalar resource identifiers:

  1. In a browser, go the Microsoft Application Registration Portal and log in with the Microsoft Azure account that was used to register the Xcalar Compute Engine and Xcalar Design services.

    The My applications list appears.

  2. Locate and select the Xcalar Compute Engine unique application name you provided when registering Xcalar. For example, Production Xcalar Instance.

    3. The Xcalar Compute Engine unique application name Registration page appears.

  3. From the Platforms section, record the Application ID URI value.
  4. Go back to the My applications page by clicking the My applications link.

    5. The My applications list appears.

  5. Locate and select the Xcalar Design unique application name you provided when registering Xcalar. For example, Production Xcalar Instance Client.

    6. The Xcalar Design unique application name Registration page appears.

  6. From the Properties section, record the Application ID.
  7. Log out of the Microsoft Application Registration Portal.

Enabling Azure Active Directory Authentication

To enable Azure Active Directory Authentication:

  1. Verify that Xcalar Compute Engine and Xcalar Design were registered during the Xcalar installation and that the Application ID URI and Application ID values were recorded.

  2. Log in to Xcalar Design as either the Xcalar Admin user or the default admin user.

  3. From the sidebar, click the System icon and then click the Administration icon .
  4. From the Administration panel, select Login Configuration.

    5. The LOGIN CONFIGURATION modal window appears.

  5. Select the ENABLE AZURE V2.0 COMMON AUTHENTICATION check box.
  6. In the CLIENT ID field, enter the Application ID that you recorded from the Azure Application Registration Portal. For example, 3e956d1c-ac0d-4381-a755-9cc63b7d50hb.

  7. In the XCALAR USER SCOPE field, enter the Application ID URI that you recorded from the Azure Application Registration Portal, using the following syntax:

    api://Application ID URI/access_as_user

    8. For example: api://3e956d1c-ac0d-4381-a755-9cc63b7c6dfh/access_as_user

    Where:

    access_as_user is the access name you gave for the Xcalar user.

  8. In the XCALAR ADMIN SCOPE field, enter the Application ID URI that you recorded from Azure, using the following syntax:

    9. api://Application ID URI/access_as_admin

    For example: api://3e956d1c-ac0d-4381-a755-9cc63b7c6dfh/access_as_admin

    Where:

    access_as_admin is the access name you gave for the Xcalar admin.

  9. (Optional) In the AZURE SCOPES field, enter the identification number of other protected resources that require Xcalar access.

  10. (Optional) B2C fields have been provided for B2C services authentication to access services, such as Web API, social media, and local accounts. For more information on how to set up and register B2C authentication, see the Microsoft Azure B2C authentication documentation. For more information on using the B2C authentication fields, contact Xcalar support.

    NOTE: When using B2C you must select the ENABLE AZURE B2C check box.
  11. Click Confirm & Save.

Understanding the default admin user account

Regardless of the user authentication method, you must set up an administrator user account before you can log in to Xcalar Design as an admin user. This user account, created during the Xcalar installation, provides additional privileges, such as access to the Administration panel.

The default admin user account, whose credentials are stored locally on your cluster, is a single user administrator account configured outside of an authentication service, such as LDAP or Azure AD. This account enables you to log in to Xcalar Design if LDAP and Azure AD are not available or are disabled. After logging in as the default admin user, you can enable another authentication method or reconfigure LDAP or Azure AD to resolve problems.

Creating the default admin user account

To create the default admin user account, follow these steps:

  1. Select the ENABLE DEFAULT ADMIN USER ACCOUNT check box.

  2. Enter the following information:

    • User name.
    • User password, which must consist of at least one non-blank character.
    • Email address.
  3. Click Confirm & Save.

Changing the default admin user account information

For an existing default admin user account, if you need to change the account information, follow these steps:

  1. If the default admin user account is not enabled, select the ENABLE DEFAULT ADMIN USER ACCOUNT check box.

  2. Change the user name, user password, or email address.

    NOTE: You can change the password without providing the existing password. This step is useful, for example, if you forget your password and want to reset it.
  3. Click Confirm & Save.
