Getting support from Xcalar

You can contact Xcalar for technical support through Xcalar Design. This topic describes how to obtain technical support by filing a support ticket and by starting a live chat. You can follow the steps in this topic even when Xcalar is not up and running.

Understanding the support ticket

You can send a support ticket to report a problem or to send a comment to Xcalar. To send a ticket from the cluster, the cluster must have Internet connection.

Information in a support ticket

A support ticket includes the following types of information:

  • Type of issue
  • Comment
  • License key
  • Workbook name
  • Number of nodes
  • Status
  • Node information, which includes:
    • CPU usage in percentage
    • Memory used in percentage
    • Memory used in bytes
    • Number of bytes received from the network
    • Number of bytes sent to the network
    • Node ID
    • Memory available in bytes
  • User name
  • User unique ID
  • Xcalar version

Information in a support bundle

If you report a performance issue or a crash, you can include a support bundle, which contains additional information.

A support bundle includes the following types of information:

  • Any core dumps caused by a crash.
  • Xcalar configuration (/etc/xcalar/*) and Xcalar-generated data (/var/opt/xcalar/*).
  • System statistics and configuration, such as the number of CPUs, memory, and packages installed.
  • System and Xcalar logs, if permissions allow, such as those found in /var/log/messages, /var/log/httpd/*, /var/log/Xcalar*.log, or /var/log/xcalar/*.
NOTE: The exact information included in the support bundle may vary, depending on the access rights given to Xcalar by you. By default, Xcalar does not run as the root user, therefore the file permissions set by your system administrator may prevent some information from being sent to Xcalar. For example, by default, Xcalar cannot collect system information unless it has permission to read the syslog file.

Generating a support ticket

To generate a support ticket, follow these steps:

  1. In the Xcalar Design menu, click the User icon and select File Support Ticket.
    The File Support Ticket modal window is displayed.

  2. In the Type of issue list, select whether you are generating a new ticket or modifying an existing ticket.

Do one of the following:

  • For an existing ticket:
    1. Either, from the Select Ticket table that displays all your submitted tickets, select the ticket that requires changing or in the enter an existing ticket field, enter the existing ticket identification number.
    2. Click Update Ticket.
  • For a new ticket:
    1. From the Severity list, select the severity level of the issue.
    2. From the Additional (new) information section, in the Subject field, enter a short description that identifies your issue and then in the Comments field, enter more information about the issue.
    3. (Optional) Include additional information about the issue by checking the support bundle check box.
      4. NOTE: Generating a support bundle may take a long time.
    4. Click File Ticket.
    TIP: During the support ticket generation, the File Support Ticket icon turns gray. After the ticket is generated, the icon resumes its original white color.

What happens if a support ticket cannot be sent

The most common reason for failing to send a support ticket to Xcalar is that the Xcalar cluster is not connected to the Internet. If your ticket includes the support bundle, the bundle stays on the cluster until the Internet connection is available. Xcalar automatically sends the support bundle as soon as a connection to the Internet is established.

The contents of the support ticket, excluding the support bundle, can be downloaded when Xcalar fails to send the ticket. When an error message indicates that you can download the ticket, manually send the ticket to Xcalar.

To manually send a support ticket:

  1. In the File Support Ticket modal window, click Download, which replaces Generate Ticket. A ticket file is created and downloaded to your computer.
  2. Upload the ticket file from your computer by using the Submit a request link on the portal.

Starting live chat with Xcalar technical support

To start a live chat with Xcalar technical support, follow these steps:

  1. In the Xcalar Design menu, click the User icon and select Live Chat.
    The Live Chat modal window is displayed.
  2. In the Name and Email fields, enter your name and email address. A chat log is automatically sent to the email address after the chat is finished.
  3. Click START and wait for a Xcalar technical support engineer to respond. A support ticket is generated as soon as the Xcalar technical support engineer starts the chat with you. The ticket ID is saved to your customer portal, which you can refer to later if necessary.
TIP: At any time during the chat, you can click the icon in the left corner of the Live Chat modal window title bar to send the chat log to your email address.
