Publishing a result set to a table

To publish a result set in a dataflow to a table:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

  2. From the sidebar, click the Dataflow Panel icon.

    The Dataflow Mode workspace opens.

  3. Click the Out category of operations in the Dataflow Mode toolbar.

  4. Drag the Publish Table Node to the dataflow canvas.
  5. Locate the Node with the result set that you want to publish, and connect the Node's output to the Publish Table Node.

  6. Right click the Publish Table Node and select Configure.

    The Publish Table configuration panel opens up.

  7. Enter a new table name in the Published Table Name field.
  8. By default the No Primary Key check box is unchecked. If you want to publish the table without a primary key, check this check box.
  9. If the No Primary Key check box is unchecked, select a column name from the Primary Key column drop-down.

  10. If you want to add another column to the Primary Key, click Add Another Column. This adds an empty column to the Primary Key. Click on it to select another column name to add to the Primary Key.

  11. If you want to add more columns to the Primary Key, repeat the previous step.
  12. (Optional) Leave the IMD Operator field empty. If you need to fill this field, do it under the guidance of Xcalar Support.
  13. In the Columns in Table section, select the columns you want to publish to the table.
  14. (Optional) Check Select All to select all columns.

  15. Click Save.

    This saves the configuration of the Publish Table Node and publishes the table on execution.
