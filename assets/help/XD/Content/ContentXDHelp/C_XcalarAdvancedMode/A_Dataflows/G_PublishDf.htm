Publishing a dataflow

After building a dataflow, you can publish it so that other users on the same cluster can access your dataflow. Publishing a dataflow enables you to easily collaborate with others on a dataflow, because it does not require the transfer of a dataflow file from one user to another. This topic lists the tasks you can and cannot perform on a published dataflow. It also describes the procedure for publishing a dataflow.

Tasks you can perform on a published dataflow

After a dataflow is published, all users, including the creator of the dataflow, have the same access rights to it. The tasks you can perform on a published dataflow

  • Viewing the published dataflow.
  • Copying the published dataflow.
  • Deleting the published dataflow.
  • Configuring Nodes.
  • Resetting Nodes.
  • Executing Nodes of the published dataflows, if the dataflow uses a dataset to which you have access.

Tasks you cannot perform on a published dataflow

The tasks you cannot do with a published dataflow:

  • Renaming the published dataflow.

  • Editing the published dataflow. For example, you cannot perform these editing tasks:

    • Configuring the links
    • Renaming Nodes
    • Adding, deleting, or moving Nodes
  • Publishing the published dataflow.
  • Automatically aligning or manually repositioning the Nodes.

Publishing a dataflow

To publish a dataflow:

  1. Display the dataflow you want to publish. It must be a dataflow not in the Published folder.

  2. Hover over the name of the dataflow in the Dataflows panel. Then click the publish icon, as shown in the following screenshot:

  3. In the Publish Dataflow modal window, specify the published dataflow's name using one of the following methods:

    • Enter the published dataflow's name after /Published/ in the Destination dataflow name field.
    • Click Browse. Navigate to the desired location in the Published folder. Type the name in the Save As field, and then click Save.

    With either method, you can create sub-folders in the path to the published dataflow. For example, you can enter user1/airlines/2001/df1 in the Destination dataflow name field. The dataflow named df1 will be published in the Published/user1/airlines/2001 folder.

  4. (Optional) If you want datasets used in the dataflow to be shared, click the Share datasets used in the dataflow check box. This ensures that the user accessing the published dataflow can execute the dataset Node in the dataflow even if that user does not own a dataset by the same name.
  5. Click Confirm. The dataflow is placed in the Published folder, as shown in the Dataflows panel and is available for other users to use.
  • For information about how to use a published dataflow, see Using a published dataflow.
